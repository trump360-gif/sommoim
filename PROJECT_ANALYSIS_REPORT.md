# 소모임(Sommoim) 프로젝트 분석 리포트

**작성일:** 2024년 12월 4일
**분석 범위:** 현재 구현 상태 vs 실제 소모임 앱 기능 비교

---

## 📊 Executive Summary

| 항목 | 현재 상태 | 목표 |
|-----|----------|------|
| 핵심 기능 완성도 | 75% | 100% |
| 사용자 경험 | 60% | 90% |
| 차별화 기능 | 40% | 80% |
| 운영 도구 | 70% | 90% |

**총평:** 기본적인 모임 CRUD, 참가 관리, 리뷰, 채팅 등 핵심 기능은 구현되어 있으나, 실제 소모임 앱과 비교했을 때 **사용자 유입/유지를 위한 기능**과 **오프라인 모임 활성화 도구**가 부족함.

## 3. 🔴 Critical - 반드시 추가해야 할 기능

### 3.1 정모(정기모임) & 번개 시스템
**현재 문제:** 일정과 모임이 1:N 관계로만 존재, 개별 모임 이벤트 개념 없음

**구현 필요:**
```
Meeting (소모임)
└── Event (정모/번개)
    ├── type: REGULAR | LIGHTNING
    ├── date, time, location
    ├── maxParticipants
    ├── participants[]
    └── chatRoom
```

**기대 효과:**
- 매주 14,000개 정모 개설 (소모임 앱 수준) 목표 가능
- 사용자 활동량 증가

### 3.2 취향 기반 추천 알고리즘
**현재 문제:** 단순 카테고리/지역 필터만 존재

**구현 필요:**
- 사용자 관심사 태그 (가입 시 선택)
- 참여 이력 기반 추천
- 협업 필터링 (비슷한 사용자가 참여한 모임)
- 인기도 점수 (참여자 수, 리뷰 평점, 활동 빈도)

**예시 API:**
```typescript
GET /api/meetings/recommended
  - 내 관심사 기반
  - 내 지역 기반
  - 최근 활동 기반
```

### 3.3 QR 출석 체크
**현재 문제:** 활동 참석 여부를 수동으로 체크

**구현 필요:**
- 모임장이 QR 코드 생성
- 참가자가 QR 스캔하여 출석 인증
- 위치 기반 검증 (선택적)
- 출석 통계 대시보드

### 3.4 반복 일정 생성
**현재 문제:** 매번 개별 일정 생성 필요

**구현 필요:**
```typescript
POST /api/meetings/:id/schedules/recurring
{
  pattern: "weekly" | "biweekly" | "monthly",
  dayOfWeek: 0-6,
  time: "19:00",
  count: 6  // 최대 6개
}
```

## 4. 🟡 Important - 사용자 경험 개선 기능

### 4.1 모임 초대 기능
- 초대 링크 생성
- 카카오톡/SMS 공유
- 초대 코드 시스템

### 4.2 운영진 시스템
```typescript
MeetingRole {
  HOST,      // 모임장
  CO_HOST,   // 공동 운영자
  STAFF,     // 스태프
  MEMBER     // 일반 회원
}
```
- 권한별 기능 분리 (일정 생성, 참가자 관리 등)

### 4.3 가입 심사 강화
- 자기소개 필수 입력
- 가입 질문 (모임별 커스텀)
- 프로필 사진 필수 옵션

### 4.4 일정별 채팅방
- 전체 모임 채팅 vs 개별 일정 채팅 분리
- 일정 참가자만 접근 가능

### 4.5 연령대/성별 제한
```typescript
MeetingRestrictions {
  minAge?: number;
  maxAge?: number;
  gender?: "ALL" | "MALE" | "FEMALE";
}
```

### 4.6 지도 연동
- 장소 검색 (카카오/네이버 지도 API)
- 지도에 위치 표시
- 길찾기 연동

## 7. UI/UX 개선사항

### 7.1 온보딩 플로우
**현재:** 회원가입 → 바로 홈
**개선:** 회원가입 → 관심사 선택 → 지역 설정 → 추천 모임 → 홈

### 7.2 모임 상세 페이지
**현재:** 탭 기반 (정보/활동/리뷰)
**개선:**
- 히어로 섹션 강화 (이미지, 핵심 정보)
- 다음 일정 하이라이트
- 참가 CTA 버튼 고정
- 최근 활동 프리뷰

### 7.3 홈 화면
**현재:** 섹션 기반 (배너, 카테고리, 추천)
**개선:**
- 개인화된 피드
- 내 주변 모임 지도 뷰
- 오늘의 정모/번개

### 7.4 마이페이지
**현재:** 기본 정보 + 참여 모임
**개선:**
- 활동 통계 (참여 횟수, 개설 모임 등)
- 배지/업적
- 캘린더 뷰 강화

## 9. 결론

### 강점
- 견고한 기술 스택 (NestJS + Next.js 14)
- 핵심 CRUD 기능 완성
- 관리자 도구 충실
- 실시간 채팅 구현

### 약점
- 오프라인 모임 활성화 도구 부족
- 개인화/추천 기능 없음
- 모임 내 소통 채널 단순
- 사용자 유입 동기 부족

### 핵심 제안
1. **정모/번개 시스템**은 가장 시급 - 이것이 소모임 앱의 핵심
2. **QR 출석**과 **반복 일정**은 운영 효율성에 필수
3. **추천 알고리즘**은 사용자 유지율에 직결
4. **소셜 로그인**은 가입 허들 낮추기에 효과적
